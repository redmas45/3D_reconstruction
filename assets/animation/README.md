# Humanoid motion asset

`humanoid_motion_library.blend` is the runtime asset used for supported and
plausible people. It contains:

- collection `FOR3D_Humanoid`
- one armature named `FOR3D_Rig`
- actions `FOR3D_idle`, `FOR3D_walk`, `FOR3D_brisk_walk`, and `FOR3D_run`

The repository includes a deterministic, low-poly starter asset generated by
`blender/create_motion_asset.py`. It lets local and Colab runs work without a
network request, Adobe login, or judge-time asset generation.

## Rebuild the starter asset

```powershell
& "C:\Program Files\Blender Foundation\Blender 4.5\blender.exe" `
  --background `
  --python blender/create_motion_asset.py `
  -- `
  --output assets/animation/humanoid_motion_library.blend
```

## Package downloaded Mixamo clips

Adobe requires the character and clips to be downloaded through the user's
Mixamo account. Keep those source FBX files outside Git. Download one rigged
character and the four animations for that same character/skeleton. Enable
**In Place** for every locomotion clip because the validated reconstruction
path owns world-space movement; embedded forward root motion would otherwise
move the actor twice. Then run:

```powershell
& "C:\Program Files\Blender Foundation\Blender 4.5\blender.exe" `
  --background `
  --python scripts/prepare_mixamo_motion_library.py `
  -- `
  --character C:\path\character.fbx `
  --clip idle=C:\path\idle.fbx `
  --clip walk=C:\path\walk.fbx `
  --clip brisk_walk=C:\path\brisk_walk.fbx `
  --clip run=C:\path\run.fbx `
  --output assets/animation/humanoid_motion_library.blend
```

The packager verifies that all clips use the character's skeleton, renames the
actions to the runtime contract, and produces one self-contained `.blend`.
Rendering never calls Mixamo or another cloud animation service.
